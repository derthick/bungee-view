Bungee View Tips and Tricks

Customize Bungee View
The mode menu (initially “Beginner Mode”) just under the “Bungee View” label lets you choose multiple display and behavior options. A bug on Macs hides the OK/Cancel buttons at the bottom of the Custom menu, so it is useless except to see what options you’ll get if you choose “Expert Mode.”

Notice the Mouse Documentation Line at the bottom of the header
As you move the mouse, it tells you what will happen if you click. In Expert Mode if the mouse is not over anything you can click on, it tells you what the arrow keys will do.

How to determine what filters are in effect
There is an English-like description of your current query above the buttons in the header. If it doesn’t fit on one line, you can move the mouse over it and see the full description. Each filter term is also highlighted in bright green, both along the left edge of the Tag Wall column, as well as when it appears in the Selected Result column. Since the selected result satisfies all the filters, all filter terms will all appear there (unless in Expert Mode you have selected multiple tags in one category, or selected a tag’s brown (prohibit) checkbox).

Navigate in the Selected Result column
You can add and remove filters by clicking on tags for the selected result just like you can on the bars and labels on the Tag Wall. Since you can see all the categories and tags at once, this is faster for the common case where the selected result has the tags you are interested in.

Navigate in the Results column (Expert Mode only)
After you click on a thumbnail in the Results column, you can use the arrow keys to select adjacent thumbnails.

Navigate in the Tag Wall (Expert Mode only)
After you click on a bar or label on the Tag Wall, you can use the right and left arrow keys to select adjacent tags. This is especially useful when there are too many tags to fit on the screen. You can get a list of all tags in the selected category by clicking on the category name.

Filter on multiple tags for a category (Expert Mode only)
Use control+click to add (or remove) additional single tags, or shift+click to add a range of tags to a filter on a category. The filter will match items that have at least one of the selected tags for that category. Clicking on the light green checkbox in front of a tag label has the same effect as control+clicking. Clicking on the light brown checkbox filters out items with that tag. Control+a selects all tags. Alt+click displays a row of bars for a tag without changing the filters.

Control how your browser shows help pages and result details
Bungee View tells your default browser what web page to display, but the browser decides whether to replace the current page, open a new window, or open a new tab. Some browsers let you customize this behavior. For instance, in Firefox go to Tools>Options>Tabs.

Layout Customization (Expert Mode only)
The initial text size is set to match that of the browser that launched Bungee View. You can change it interactively by dragging the header border up and down. Move the mouse over the border until you see a horizontal white line, and then drag the line. You can also adjust the widths of the Tag Wall, Matches, and Selected Result columns by dragging their borders. (If you resize the whole window, it will compute new column widths and lose your changes.) To change the number of columns of thumbnails, move the mouse to the right of the first thumbnail column, and then drag that border.

Get rid of those annoying pop-ups
Typing control+P (for Pop-up) toggles between showing tool tips as pop-ups and in the mouse documentation line in the header.

Find the most relevant tags to your query (Expert Mode only)
Tool tips display P-values for the difference between observed and expected Tag counts, and between observed and expected median values. A P-value (e.g. “(p=0.07)”) is the probability that the observed value (or a more extreme value) would occur under the null hypothesis that each case is assigned a value by sampling from the expected distribution. The displayed P-values are Bonferroni corrected by multiplying the raw value by the number of displayed bars. Corrected P-values < 0.01 are considered significant. Tags with such low P-values are colored green or brown, depending on whether the association with the query is positive or negative.
